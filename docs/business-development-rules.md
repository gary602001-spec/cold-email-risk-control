# 外贸冷邮件开发业务总规则

这份文档是日常开发客户时的总规则。目标不是多发邮件，而是稳定地找到相关客户，用可信身份发出像真人写的业务邮件，并且在风险出现时及时停止。

## 1. 总判断

每次准备发邮件、导入名单、提高发送量、写跟进、处理回复或排查异常时，先做判断：

```text
评级：S / A / B / C / 禁止发送
动作：可发 / 小改后可发 / 减量测试 / 暂停发送 / 禁止发送 / 需要先确认合规
```

必须同时回答：

- 现在能不能做。
- 现在不能做什么。
- 特别注意什么。
- 应该规避什么。
- 如果要继续，先改哪几件事。

## 2. S/A/B/C 客户评级

### S 级

高度优先开发。

条件：

- 公司真实存在。
- 行业和产品高度匹配。
- 有明确采购、进口、渠道、产品线、展会、询盘或市场证据。
- 联系人职位直接相关，例如采购、老板、进口、运营、产品经理、品类负责人。
- 使用公司邮箱。
- 国家/地区合规风险可控。
- 邮箱认证、退订、签名、官网、节奏都已准备好。

动作：可以低量、稳定、个性化发送。

### A 级

可以开发，但先小改。

条件：

- 公司和行业匹配。
- 联系人基本相关。
- 至少有一个具体触发点。
- 名单来源清楚。
- 没有退订、拒绝、投诉记录。

动作：小改内容后发送，优先写清楚“为什么联系你”。

### B 级

谨慎测试。

条件：

- 公司真实。
- 行业可能相关，但采购信号弱。
- 联系人不一定是负责人。
- 合规依据、国家/地区或名单来源还需要补充。

动作：先补资料；如果要发，只能极小量测试，不进入常规序列。

### C 级

默认不发。

条件：

- 公司真实，但产品相关性弱。
- 联系人泛泛。
- 没有明确触发点。
- 名单来源一般。

动作：不进入冷邮件发送，除非补充证据后升到 A/B 级。

### 禁止发送

命中任一项就拦截：

- 买来的大包名单。
- 爬来的不相关邮箱。
- 私人邮箱乱发。
- 无法说明来源、获取时间、业务相关理由。
- 对方已拒绝、退订、投诉或要求停止联系。
- SPF、DKIM、DMARC 实发不通过。
- 无退订或停止联系机制。
- 加拿大、澳大利亚、欧盟、英国等市场合法依据不清。
- 标题误导或正文像广告群发。

## 3. 发信基础设施规则

必须做到：

- 用公司域名邮箱，不用个人 Gmail、QQ、163 做外贸冷发。
- SPF、DKIM、DMARC 配好。
- 用测试邮件确认 `SPF=pass`、`DKIM=pass`、`DMARC=pass`。
- From 域名与 SPF 或 DKIM 对齐。
- PTR/反向 DNS、TLS、Message-ID 正常。
- From、Reply-To、签名、官网、品牌一致。
- 官网可打开，产品页真实。
- 美国商业邮件有有效实体邮寄地址。
- 批量或营销邮件尽量支持 `List-Unsubscribe` 和一键退订。

不能做：

- 新域名或新邮箱一上来大量发。
- 认证不通过还继续发。
- 频繁换域名、换发件人、换 IP、换模板。
- 用假暖箱、假打开、假回复、互相标记非垃圾。

## 4. 名单规则

每个联系人至少要能说明：

- 这家公司是谁。
- 为什么它可能需要你的产品。
- 这个联系人为什么可能负责或影响采购。
- 邮箱从哪里来。
- 什么时候获取的。
- 属于哪个国家/地区。
- 是否有合法依据或合理业务相关性。
- 是否曾退订、拒绝、投诉。

名单来源要单独评分：

| 指标 | 用途 |
| --- | --- |
| 硬退信率 | 判断邮箱质量 |
| 投诉率 | 判断相关性和接受度 |
| 退订率 | 判断打扰程度 |
| 总回复率 | 只做参考 |
| 正向回复率 | 判断名单价值 |
| 有效对话率 | 判断是否进入业务沟通 |
| 询盘数量 | 判断商业价值 |

某个来源硬退信高、投诉高、退订高、正向回复低时，暂停该来源，不要只靠改模板继续发。

## 5. 第一封邮件规则

第一封邮件目标不是成交，而是确认相关性和负责人。

必须包含：

1. 为什么联系对方。
2. 你是谁。
3. 一句话说明能帮什么。
4. 一个或两个可信证据。
5. 一个轻动作，例如是否负责采购、是否方便发目录。
6. 一个退出口，例如不合适可回复告知，不再打扰。

第一封至少要有一个触发点：

- 对方网站产品线。
- 对方客户类型。
- 对方销售渠道。
- 对方进口或采购信号。
- 展会、询盘、名片或已有业务接触。
- 与对方品类明确匹配的产品点。

不能写：

- 大段公司介绍。
- 十几张图片。
- 大附件。
- 多个链接。
- 短链接或不透明跳转链接。
- `best price`、`top quality`、`hot sale`、`urgent` 这类促销词堆叠。
- 假装 `Re:`、`Fwd:`、订单、付款、紧急事项。

## 6. 跟进规则

最多 3-4 封。

每封必须新增信息：

- 第 1 封：简短介绍和轻动作。
- 第 2 封：补一个与对方品类相关的价值点。
- 第 3 封：补一个认证、出口市场、客户类型、产能或案例。
- 第 4 封：确认是否找对负责人，礼貌收尾，不再打扰。

不能做：

- 每天追。
- 换标题装作新邮件。
- 多人轮流轰炸同一家公司。
- 对方不感兴趣还继续发。
- 只写 `just following up`、`any update`、`did you see my email`。

## 7. 回复处理 SOP

- 不需要、不感兴趣、停止联系：立即停止，进全局黑名单。
- 退订、投诉、abuse：立即停止，进全局黑名单，不换邮箱继续发。
- 以后再说：设置未来提醒，不继续当前序列。
- 问价格：先确认规格、数量、市场、用途，再报价。
- 要目录：发轻量资料或官网目录页，不塞大附件。
- 转给同事：只联系被转的人，原联系人停止追发。
- 自动回复离职、休假、邮箱停用：更新联系人状态，不继续追发。

## 8. 发送节奏规则

原则：慢启动、稳定、分散、可控。

必须做到：

- 新邮箱先正常使用一段时间。
- 刚开始低量发送。
- 每天稳定，不暴增。
- 分批发送，不短时间密集发送。
- 同一模板不要大规模复制。

硬停止：

- 投诉率目标低于 `0.1%`。
- 投诉率接近或达到 `0.3%`：暂停。
- 硬退信率超过 `2%`：暂停该名单来源并清洗。
- 硬退信率超过 `5%`：停止本批次。
- 退信、投诉、延迟、拒收异常：先暂停或减量，不加量。
- 负面文字反馈集中出现：先暂停，即使后台投诉还没显示。

负面文字反馈包括：

- `remove me`
- `stop emailing`
- `spam`
- `not relevant`
- `how did you get my email`
- `report abuse`

## 9. 合规底线

保守默认：

- 美国：注意 CAN-SPAM，真实身份、有效实体地址、清楚退订。
- 欧盟：注意 GDPR/ePrivacy，个人姓名公司邮箱也可能是个人数据。
- 英国：注意 PECR / UK GDPR，区分公司主体和个体经营者等对象。
- 加拿大：CASL 更严格，无同意或可证明业务关系时先确认合规。
- 澳大利亚：Spam Act 要求同意、识别发件人、有效退订，无明确依据时先确认合规。

合规信息不足时，不给 `S` 或 `A`，也不给 `可发`。

## 10. 每天监控

每天看：

- 硬退信率。
- 软退信率。
- 投诉率。
- 退订率。
- 总回复率。
- 正向回复率。
- 有效对话率。
- 询盘数量。
- 域名/IP 声誉。
- Gmail、Outlook、Yahoo 延迟或拒收。
- 哪些模板容易出问题。
- 哪些名单来源质量差。

## 11. 发送前 30 秒检查

发送前问自己：

1. 这个人为什么可能需要我的产品？
2. 我能说清楚为什么联系他吗？
3. 标题有没有误导？
4. 正文像不像真人写的？
5. 有没有退订或停止联系出口？
6. 如果他截图投诉，这封邮件站得住吗？
7. 这个邮箱和域名认证通过了吗？
8. 对方如果拒绝，我是否会立刻停止？

有一个答不上来，就不要发。

## 12. Codex 使用方式

以后把准备做的动作发给 Codex，例如：

- 我要发这一批，帮我评级。
- 这封第一封开发信能发吗？
- 这个名单来源能不能导入？
- 今天想从 50 封加到 100 封，可以吗？
- 对方这样回复，我下一步怎么做？
- 退信率升高了，帮我判断要不要停。

Codex 必须输出：

```text
评级：S / A / B / C / 禁止发送
动作：可发 / 小改后可发 / 减量测试 / 暂停发送 / 禁止发送 / 需要先确认合规

主要风险：

必须先改：

建议执行：

现在不能做：

特别注意：

应该规避：
```

核心标准：宁可少发，也不要伤域名、伤品牌、骚扰不相关的人。
