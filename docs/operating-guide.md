# 日常操作指南

这个指南说明如何把本项目当作发信前风控台使用。

## 新邮箱上线前

使用 [templates/pre-send-risk-review.md](../templates/pre-send-risk-review.md)，重点填写：

- 发信域名。
- From 邮箱。
- Reply-To 邮箱。
- SPF、DKIM、DMARC。
- 测试邮件头认证结果。
- PTR/反向 DNS。
- TLS。
- List-Unsubscribe 和一键退订。
- 官网、签名、电话、实体地址。

没有实发认证结果时，不要给 `可发` 结论。

## 导入名单前

重点填写：

- 名单来源。
- 获取日期。
- 国家/地区。
- 客户是否真实存在。
- 业务相关理由。
- 联系人职位。
- 合法依据。
- 是否有同意、询盘、展会、既有业务关系或其他证据。
- 是否排除退信、退订、拒绝、投诉联系人。

如果名单来源不清，默认高风险。

## 审核第一封邮件

使用 [templates/email-copy-review.md](../templates/email-copy-review.md)。

重点看：

- 标题是否误导。
- 正文是否过度营销。
- 是否太长。
- 是否堆图片、附件、链接。
- 是否有停止联系说明。
- 美国客户是否有实体邮寄地址。
- 批量或营销邮件是否有退订头和正文退订方式。

## 提高发送量前

先看过去 7 天：

- 每天发送量是否稳定。
- 硬退信率。
- 软退信率。
- 投诉率。
- 退订率。
- 回复率。
- 正向回复率。
- 是否有 Gmail、Outlook、Yahoo 延迟或拒收。

有异常时先暂停或减量，不加量。

## 出现异常时

使用 [templates/sending-incident-review.md](../templates/sending-incident-review.md)。

恢复发送前必须确认：

- SPF、DKIM、DMARC 实发通过。
- From 域名对齐。
- PTR、TLS、Message-ID 正常。
- 硬退信地址已移除。
- 退订、投诉、abuse 已进全局黑名单。
- 问题名单来源已暂停。
- 问题模板已修改。

恢复时只小量测试，不直接恢复原发送量。

## 推荐审核输出

```text
评级：B
动作：暂停发送

主要风险：
1. 测试邮件头 DMARC 未通过。
2. 名单来源无法说明获取日期和业务相关理由。
3. 没有退订或停止联系机制。

必须先改：
1. 修复 SPF/DKIM/DMARC 并重新测试邮件头。
2. 补名单来源、国家、职位、业务相关理由。
3. 加入停止联系说明和黑名单处理流程。

建议执行：
1. 暂停本批次。
2. 清洗名单。
3. 用 20-50 封小批量恢复测试。

现在不能做：
1. 不能继续加量。
2. 不能把这批名单直接进入常规序列。

特别注意：
1. 先看实发邮件头，不只看 DNS 记录。
2. 负面回复要按退订处理。

应该规避：
1. 假暖箱。
2. 空泛跟进。
```
