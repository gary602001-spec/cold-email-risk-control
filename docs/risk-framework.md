# 完整风控框架

本框架用于外贸冷邮件发信前审核。核心判断不是“能不能多发”，而是这次发送是否像一个真实、低投诉、可退出的正常商务联系。

每次审核先给 `S/A/B/C/禁止发送` 评级，再给动作：`可发`、`小改后可发`、`减量测试`、`暂停发送`、`禁止发送`、`需要先确认合规`。

详见 [外贸冷邮件开发业务总规则](business-development-rules.md)。

## 1. 发信基础设施

必须检查：

- 使用公司域名邮箱，不使用个人 Gmail、QQ、163 等邮箱做冷发。
- SPF、DKIM、DMARC 已配置。
- 不能只看 DNS 记录存在，要用测试邮件确认 `SPF=pass`、`DKIM=pass`、`DMARC=pass`。
- 可见 `From` 域名与 SPF 或 DKIM 域名对齐。
- 自建或专用发信源要有有效 PTR/反向 DNS。
- TLS 正常。
- `Message-ID` 正常、稳定、符合邮件格式规范。
- `From`、`Reply-To`、签名、官网、品牌一致。
- 官网可访问，产品页真实，不像空壳公司。

红线：

- DNS 未配好就发。
- 邮件头认证不通过还继续发。
- 新域名或新邮箱第一天大量发送。
- 频繁换域名、换发件人、换 IP、换模板。

## 2. 发件身份

必须检查：

- 真实姓名。
- 真实职位。
- 真实公司。
- 真实官网。
- 真实联系电话。
- 清楚的公司签名。
- 美国商业邮件应包含有效实体邮寄地址。

红线：

- 发件人身份模糊。
- 显示名误导收件人。
- From 邮箱和签名品牌不一致。
- 标题或展示名伪装成订单、付款、回复或内部沟通。

## 3. 名单质量

只发给有业务相关性的联系人。

客户先分层：

- `S 级`：高度匹配，有明确采购或业务信号，可以低量稳定发送。
- `A 级`：匹配度高，有触发点，小改后发送。
- `B 级`：可能匹配但证据弱，先补资料或极小量测试。
- `C 级`：公司真实但相关性弱，默认不发。
- `禁止发送`：来源不清、无相关性、私人邮箱、已退订/拒绝/投诉或合规依据不足。

必须检查：

- 客户公司真实存在。
- 对方确实可能采购或影响采购。
- 联系人职位相关，例如采购、老板、进口、运营、产品、品类负责人。
- 优先公司邮箱。
- 每家公司联系人数量可控。
- 退信、退订、拒绝、投诉联系人已移除。

名单记录建议字段：

- 来源。
- 获取日期。
- 国家/地区。
- 公司名称。
- 联系人职位。
- 业务相关理由。
- 公司邮箱或私人邮箱。
- 是否角色邮箱、通用邮箱或 catch-all。
- 合法依据。
- 同意、询盘、展会、既有业务关系或其他证据。
- 首次联系日期。
- 最近联系日期。
- 回复、退订、拒绝、投诉状态。

红线：

- 买来的大包邮箱名单。
- 爬来的不相关邮箱。
- 私人邮箱乱发。
- 同一公司多人同时群发。
- 名单无法说明来源、获取时间、业务相关理由或合法依据。
- C 级名单进入常规发送。

## 4. 发送节奏

原则：慢启动、稳定、分散、可控。

建议检查：

- 新邮箱先有一段真实业务使用。
- 刚开始低量发送。
- 每天稳定，不暴增。
- 分批发送，不在短时间内密集发送。
- 同一模板不要大规模复制。
- 退信、投诉、退订异常时立即减量或暂停。

硬停止阈值：

- 垃圾投诉率目标尽量低于 `0.1%`。
- 投诉率接近或达到 `0.3%` 时暂停排查。
- 硬退信率超过 `2%` 时暂停该名单来源并清洗。
- 硬退信率超过 `5%` 时停止本批次。
- 任意硬退信地址立即移除。
- 连续软退信地址先暂停，不继续追发。

## 5. 邮件内容

第一封邮件应短、准、像真人写的业务沟通。

第一封必须有一个具体触发点，例如对方网站产品线、客户类型、采购场景、进口记录、展会接触、询盘来源或明确业务匹配点。

建议结构：

1. 说明为什么联系对方。
2. 说明你是谁。
3. 用一句话说明能帮什么。
4. 给一个或两个可信证据，例如出口国家、认证、客户类型、产能、案例。
5. 给轻动作，例如问是否负责采购，或是否方便发目录。
6. 给退出口，例如不合适可回复告知，不再打扰。

避免：

- 大段公司介绍。
- 十几张图片。
- 大附件。
- 多个链接。
- 过度 HTML 装修。
- 短链接或不透明跳转链接。
- `best price`、`top quality`、`hot sale`、`urgent` 等过度促销词。

## 6. 标题

标题要具体、业务相关、不过度营销。

可以参考：

- `Supplier for [product]`
- `[Product] for [customer company/type]`
- `Question about your [category] sourcing`
- `Factory support for [product line]`
- `Can we send you [catalog/sample info]?`

避免：

- 假装 `Re:` 或 `Fwd:`。
- `Urgent!!!`
- `Important order`
- `Payment`
- 全大写。
- 感叹号堆叠。
- 和正文不一致的标题。

## 7. 退出机制

最低要求：

- 正文里有停止联系说明。
- 对方回复不感兴趣，立即停止。
- 对方退订，立即加入黑名单。
- 投诉、abuse、停止联系请求按退订处理。
- 不换邮箱继续发同一个人。

批量或营销邮件建议：

- 设置 `List-Unsubscribe`。
- 支持一键退订。
- 退订处理不要求登录、不要求额外提供个人信息。

## 8. 合规边界

不同地区规则不同。本项目采用保守处理：

- 美国：注意 CAN-SPAM，包括真实发件信息、不误导标题、有效实体邮寄地址、清楚退订。
- 欧盟：注意 GDPR、ePrivacy 和各成员国规则；公司邮箱中含个人姓名也可能是个人数据。
- 英国：注意 PECR / UK GDPR；B2B 也要区分公司主体、个体经营者等情况。
- 加拿大：CASL 更严格，无同意或可证明业务关系时默认 `需要先确认合规`。
- 澳大利亚：Spam Act 要求同意、识别发件人、有效退订；无明确依据时默认 `需要先确认合规`。
- 其他地区：至少保留真实身份、业务相关性、退订和停止联系机制。

## 9. 跟进节奏

建议最多一个轻量序列：

1. 第 1 封：业务相关的简短介绍。
2. 第 2 封：补充一个具体价值点或案例。
3. 第 3 封：确认是否找对负责人。
4. 第 4 封：礼貌收尾，不再打扰。

原则：

- 每封之间留间隔。
- 不每天追。
- 不换标题伪装新邮件。
- 不多人轮流轰炸同一客户。
- 对方没兴趣就停。
- 打开但不回复，不代表可以无限追。
- 每封跟进必须新增信息，不写空泛 `just following up`、`any update`、`did you see my email`。

## 9.1 回复处理

- 不需要、不感兴趣、停止联系：立即停止，进全局黑名单。
- 退订、投诉、abuse：立即停止，进全局黑名单。
- 以后再说：设置未来提醒，不继续当前序列。
- 问价格：先确认规格、数量、市场、用途，再报价。
- 要目录：发轻量资料或官网目录页，不塞大附件。
- 转给同事：只联系被转的人，原联系人停止追发。
- 自动回复离职、休假、邮箱停用：更新联系人状态，不继续追发。

## 10. 监控指标

每天看：

- 硬退信率。
- 软退信率。
- 垃圾投诉率。
- 回复率。
- 正向回复率。
- 退订率。
- 域名/IP 声誉。
- Gmail、Outlook、Yahoo 是否延迟或拒收。
- 哪些模板容易进垃圾箱。
- 哪些名单来源质量差。
- 哪些来源正向回复低、退订高或负面文字反馈多。

异常处理顺序：

1. 暂停或减量。
2. 查实发认证。
3. 查退信和拒收错误。
4. 查名单来源。
5. 查内容和标题。
6. 查退订和投诉处理。
7. 小量恢复，不直接恢复到原发送量。

负面文字反馈集中出现时，即使投诉后台还没显示，也先暂停该名单来源或模板。

## 11. 禁止的做法

- 虚假暖箱。
- 自动互发邮件。
- 假打开、假回复。
- 标记非垃圾的互助网络。
- 购买大包名单。
- 标题欺骗。
- 无退出机制。
- 对拒绝、退订、投诉联系人继续发送。
- 把营销开发内容塞进订单、付款、物流、售后等交易或服务邮件里。
