# Security Policy

## 报告范围

本项目主要是文档和模板，不包含运行时代码。仍然欢迎报告以下问题：

- 文档中意外包含私人信息、密钥、客户数据或邮箱名单。
- 模板鼓励不安全或高风险操作。
- 规则可能导致绕过退订、继续骚扰或误导收件人。

## 不要提交的信息

请不要在 issue 或 PR 中提交：

- 真实客户名单。
- 私人邮箱数据。
- API key、SMTP 密码、OAuth token。
- 未脱敏的退信原文。
- 未脱敏的邮件头。

## 报告方式

如果问题不涉及敏感信息，可以直接开 issue。  
如果涉及敏感信息，请先脱敏后再提交，或只描述问题类型和影响范围。
